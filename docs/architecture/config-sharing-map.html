Plexus architecture

One local source, native files everywhere.

Plexus keeps rules, skills, and MCP servers in a canonical store, then projects them into each AI agent using the least invasive reversible mechanism for that file type.

Is this the optimal solution?

For a local-first, multi-agent workstation: yes as a default. It uses symlinks for single-purpose assets, partial writes for shared agent files, and backups before native paths are touched.

Rules targets

symlink
~/.claude/CLAUDE.md -> rules/global.md
~/.codex/AGENTS.md -> rules/global.md

Agents keep reading their official instruction filename, but that path points to the same Plexus-managed baseline.

Skill targets

dir link
~/.codex/skills/<id> -> skills/<id>
~/.claude/skills/<id> -> skills/<id>

This is the cleanest part of the system: one folder, many native locations, no format translation.

MCP targets

hybrid
shared: rewrite only mcpServers / mcp_servers
exclusive: native file -> .cache/mcp/<agent>.json

Claude Code, Codex, Gemini, and Qwen keep their private config. Cursor and Factory can safely point to Plexus-owned cache files.

Claude Code shared MCP, rule symlink, skill dir symlinks
Cursor exclusive MCP symlink, command dir symlinks
Codex TOML partial-write, AGENTS.md, skills
Gemini / Qwen JSON partial-write, native instruction file, skills
Factory Droid exclusive MCP symlink, skills
Symlink-first Best for rules and skills because each native path can point to one canonical file or folder.
Partial-write Best for shared MCP files because auth, profiles, and app state must remain untouched.
Cache symlink Best for exclusive MCP files because Plexus can own the whole file safely.
Backup first Every native write is reversible from snapshots or collision quarantine.
